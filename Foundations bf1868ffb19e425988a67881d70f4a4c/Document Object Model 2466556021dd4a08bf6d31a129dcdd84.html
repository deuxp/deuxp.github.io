The 'DOM'

When a page is loaded the browser is running a process that creates a Document object model in memory to represent all of the html code on that page. Each html element is a also called a 'Node' or 'DOM Node', not to be confused with NodJS.

Edit the DOM

  • In short the DOM is the web page once the browser has parsed the html into nodes and objects (tree like structure) and applied the css and javascript, modifying the html. The Document (html), Object (Browser Parsing → Javascript applied), Model. Once a JS file is applied to an html, it can totally alter the structure of the page.
    • In other words, HTML represents initial page content, and the DOM represents current page content. When JavaScript adds, removes, or edits nodes, the DOM becomes different than the HTML.

Editing CSS

The Styles tab on the Elements panel lists the CSS rules being applied to whatever element is currently selected in the DOM Tree.

CSS classes being applied to the selected element are displayed in the Styles tab. This tab is what you will focus on for CSS related inspection & edits.
  • These are all of the CSS rules that are currently applied to the inspected element (in-focus).
  • The top tab - elements.style - is where you can add a CSS declaration.
  • If you scroll further down you will see the box-model → you can hover to highlight the space on the element.
    • Double-Click on value of the box model to alter. Defaults → px
  • click .cls on the top right of the styles window to add a class to the focused element.. or to add the element to an existing class.
    • a new class checkbox appears. You can toggle the class.
  • :hov besides .cls → toggle force state → known as Pseudo-state
