note on multiple actions:

Something we commonly see is console.log() as an action in each branch‚Ä¶

a % 2 === 0 ? console.log(`${a} is even.`) : console.log(`${a} is odd.`);

These are both simple actions that call the same function. Now let‚Äôs consider a couple things:

  • a ternary is an expression
  • console.log() accepts any expression and parses it as text characters to the console as best it can.

That means we don‚Äôt have to call the method in each branch, we can pass the entire ternary as the argument.

console.log(a % 2 === 0 ? `${a} is even.` : `${a} is odd.`);

All expressions boil down to one value. It‚Äôs that value that will be logged in this instance. The point here is that ternary expressions should be simple if they are to be easy to read and comprehend. They are not a de facto replacement for if..else, only a convenience to lighten the code.

Lesson 4: Scope

Global scope -

  • declared outside of the block of code
  • can be accessed by any code in the program

Block Scope -

  • declared inside a block of code
  • can only be accessed within that block
  • known as local variable

Scope pollution

Avoid unintended behavior of your variables colliding:

  num = 100; // Take note of this line of code
  console.log(num);

advantages of block scoping

  • your code is more legible since your not scouring the program to know what variables you are working with.
  • everything is organized into bite sized blocks
  • easier to maintain your code since it will be modular
