Adding a directory to the $PATH

export PATH="path/to/my/executables-directory:$PATH"
  1. added our executables dir
  1. then :/
  1. then $PATH again.. this preserves the old path locations while making your directory the highest priority.
  • the path variable determines which directories are searched when command is entered.

  • $PATH is an ordered, :colon separated list of directories that contains executable files.
  • you can add to PATH so you dont have to memorize the full dir path for your commands
  • Modifications to PATH should be done in a .bashrc

HOT TIP!

use echo 'foo "bar"' >> ~/.bashrc to add lines to your bashrc file without having to open it in vim everytime

  • the single quotes makes sure to pass the whole thing as a string. otherwise trouble to be had.

Permissions

Permissions are given to files and directories; not users and groups.

Divided into 2 parts:

  1. ownership
    1. user
    1. group
    1. other
  1. access
    1. read
    1. write
    1. execute

  • based off of your user account for context.
  • can be confusing and a basic understanding goes a long way.

Interpreting Permissions

d- directory

r- read

w- write

x- execute

  • It is the files and directories that hold these privilages for the users and groups; not the users and groups being bestowed with this access.
    • defiined by the files and directories themselves
