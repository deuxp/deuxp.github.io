Associating Models

Active Record associations let you easily declare the relationship between two models. In the case of comments and articles, you could write out the relationships this way:

  • Each comment belongs to one article.
  • One article can have many comments.
class Comment < ActiveRecord::Base
  belongs_to :article
end

You'll need to edit app/models/article.rb to add the other side of the association:

class Article < ActiveRecord::Base
  has_many :comments
  validates :title, presence: true,
                    length: { minimum: 5 }
end

These 2 declarations enable a good bit of automatic behaviour.

ie,.

if you have an instance variable @article containing an article, you can retrieve all the comments belonging to that article as an array using @article.comments

in other words, the comments are accessible from the associated object as an attribute.

THen config/routes: add the following

resources :articles do
  resources :comments
end

This creates comments as a nested resource within articles. This is another part of capturing the hierarchical relationship that exists between articles and comments.
