#CLI

# start server - self updates
$ bin/rails server

# generates a new controller - add resources in the config/route file
$ bin/rails generate controller controller_name [action_name]

# `g` alias for generator - without arg shows list of generators - rspec too
$ bin/rails g

# generate the resource REST api routes - can run again to view them
$ bin/rake routes

# generate a database migration file inside the db/migrate directory
$ bin/rails generate model [col_name:type] [col_name2:type] [...]

# runs the migration
$ bin/rake db:migrate

# environment console -> can directly manipulate db with active record, so you don't need to create a form just to test the db
$ bin/rails c[onsole]

  • flow → error led
    1. Generate the controller
    1. config the root route
      • config your resources - (endpoints) makes a new controller and view associations however you must create the view file according to the expected path shown in the thrown error
    1. bin/rake routes → initializes all the possible routes for your resources
      1. You can run this again if you forget what is available when you are defining your actions in the controller
    1. Set up the new action in the controller of the new resource
      • which is literally a func with the name of the restful endpoint
    1. Create the associated view file for the new action
    1. Populate the page with the elements you want
      • opportunity to use the many helpers available
    1. Generate Models for the database → they are ruby class objects that interact with the database.
      1. Run a Migration - with bin/rake db:migrate/turninto
    1. Back in the controller, use the new model to save the data from the POST in the database.
    1. Whitelist the parameters → see strong parameters
    1. Create a Show action in the controller with the Create action
