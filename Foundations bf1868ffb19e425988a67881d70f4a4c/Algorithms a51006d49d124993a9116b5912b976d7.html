Selection Sort | O(n2)O(n^2)O(n2)﻿

Use-case

Sorting an unordered list ⇒ placing in order

Performance

Slow, since the algorithm goes through the list one by one O(n); pops out the largest into new list, then goes back to the list O(n), n times, the algo is O(n * n).

Implementation

Basically a series of 2 functions. The first to test if the number is < or >. This is then used as a callback in a function that pops the item out and puts it in a new array in this function.. return array.


const sort = function(array) {
  const newArr = [];
  while (array.length > 0) {
    const min = Math.min(...array);
    const index = array.indexOf(min);
    newArr.push(array.splice(index, 1)[0])
  }
  return newArr;
};

Breadth first Search

Its the tree traversal version of searching an unsorted array.. Its how you can search through every node in a tree.

  • start with the root node
    • move on the roots children
      • move on to those nodes children

It searches row by row. It completes the search of all the nodes closest to the root before moving on to nodes further away

Explores widely

Depth-first Search

“Depth first traversal will visit each node on an entire path, all the way out to a leaf node, before visiting nodes on the next path”

Always tries to visit the leaf nodes

Will attempt to traverse all the way to the leaf first before moving on to the next branch. that is the Search path for this one.. It eliminates a whole branch.. Visits all the nodes childeren and childrens children, before starting the search again from the next sibling of the roots children.

  • when it reaches a leaf and there are no children,
    • it searches the next sibling
      • if there is no sibling
        • search the parents sibling
          • if there is no parent sibling
            • search the ancestors sibling
              • if there is an ancestor sibling
                • search their children
                  • if there is no children
                    • search the next sibling
                      • if there is no sibling
                        • search the parents sibling

So the pattern is:

  1. search the first child node to the leaf
  1. At the leaf visit the next sibling
    1. visit children
  1. no sibling ? find the parents sibling
    1. repeat 2 and 3

so for traversal like this I reckon that it’s important to know your position on the tree as you are traversing → what is the head pointing to.
