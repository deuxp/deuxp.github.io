Clone Repo

  1. Clone or create the repo online.
    1. copy the url next to the option that you want ie., SSH

    Commands related to a remote repository:

    $ git clone git@github.com:USER-NAME/REPOSITORY-NAME.git 

    or 

    $ git clone https://github.com/user-name/repository-name.git

    or

    starting locally, you want to (a) init, (b) connect, (c) select branch and (d) push upstream.

$ git init

$ git branch -M main
$ git push -u origin main

$ git diff

git reset → removes file from staging area

Fork

  1. first fork the repo on the repo page → use the fork button on the top right
    1. This fork will show up in your github page.
  1. next clone your forked copy into your computer.
    1. it will make its own file
    1. Please do not copy into a folder that already has a git initialized

$ git clone <yer forked repo's url>

  • now you have your own forked version to play with.

You can have more than one stream → repo location (1) your fork, to work on, and (2) the original repo for pulling the latest version to work with, and to eventually push your finished feature to be reviewed or merged. Be sure to give them different names. The convention are (a) origin for your fork or your repo, and (b) upstream, for the original.

Adding the upstream remote

You will typically do this once you've clone your fork to your local. You are adding another remote connection; the upstream (original) version.

This way if the original changes or is updated, then you can pull that code and start working with the new code.

This way you are sure to make your feature work with the most recent version.

Also if you're working on a bug; maybe the updated version has already fixed it.

etc.

In your forked repo folder:

$ git remote add upstream <original repo's url>

Branch

While you are working on your feature with your forked version of the project; you push your changes up to a BRANCH of the original. You keep your changes there until they are ready and can be reviewed before they are merged into the MAIN branch.

This keeps the main stable until yours is ready to be merged.

Create

When you create a branch git copies everything from the current branch and places it in the branch that you have requested.

$ git branch <BRANCHname>
$ git checkout <BRANCHname>

$ git branch  # this lists the branches

# real-world shortcut to create a branch and switch to it

Rename your local master branch into main

git branch --move master main.
git push --set-upstream origin main.
git branch --all * 
# main 
# remotes/origin/HEAD -> origin/master 
# remotes/origin/main 
# remotes/origin/master.
git push origin --delete master

Merge

$ git checkout master

$ git merge iss53

Once your pull request has been approved and merged to the main branch of the original - you'll want to merge the updated original to your local version.

# move to the branch that you want to move into:
$ git checkout gh-pages

# Tell git what what branch you want to merge in:
$ git merge add-deuxp

# tidy up by deleting your feature now that the branch has merged
$ git branch -d add-deuxp

# Delete the branch from your fork on github
$ git push <REMOTENAME> --delete <BRANCHNAME>

Rollback

# if changes are local
$ git reset [log]

# if changes are pushed to remote and other people have them
$ git revert [log]

stash

# takes it out
git stash 
# puts it in
git stash pop

git stash drop
#deletes all stashes
git stash clear

Pull

Keeps your file up to date with updates from collaborators. Stay up to date with the latest state.

PULL any changes that have been made to the main branch.

$ git pull <remoteName> <branchName>

If nothing has changed → 'Already up-to-date'

If changes → they will merge into your local version

Pull requests

When you have finished your fork, you want to send those changes to the maintainer of the original. So everyone can benefit.

Request that they pull your changes.

→ pull request

Created from your forked repo!

  1. first go to the maintainers page and see select your branch name.
  1. contribute button on top right
  1. Submit the pull request
  1. Wait for someone to approve
