what is the difference between a web page, web site web server searrch engine

web page

A document which can be displayed in a web browser such as Firefox, Google Chrome, Opera, Microsoft Internet Explorer or Edge, or Apple's Safari. These are also often called just "pages."

website

A collection of web pages which are grouped together and usually connected together in various ways. Often called a "web site" or a "site."

web server

A computer that hosts a website on the Internet.

search engine

A web service that helps you find other web pages, such as Google, Bing, Yahoo, or DuckDuckGo. Search engines are normally accessed through a web browser (e.g. you can perform search engine searches directly in the address bar of Firefox, Chrome, etc.) or through a web page (e.g.Â bing.comÂ orÂ duckduckgo.com).

Many beginners on the web confuse search engines and browsers. Let's make it clear: A browser is a piece of software that retrieves and displays web pages; a search engine is a website that helps people find web pages from other websites.

Don't confuse the infrastructure (e.g., the browser) with the service (e.g., the search engine). The distinction will help you quite a bit, but even some professionals speak loosely, so don't feel anxious about it.

to publish a website you need either a static or dynamic web server

A static web server is also referred to as a stack. It consists of a computer (hardware) with an HTTP server (software) installed on it.

we call it static since the server sends the hosted files back to you "as-is".

A dynamic web server consists of a static web server plus extra software, most commonly an application software and a database.

it is called "dynamic" because the application software updates the hosted files before sending the content to your browser via the HTTP server.

as an example a thousand page website site like wikipedia hosts a few HTML templates which are filled in by the application server from a giant database before it is sent to your browser via the HTTP server. This method is easier to maintain and distribute the content, as opposed to thousands of static HTML documents.
