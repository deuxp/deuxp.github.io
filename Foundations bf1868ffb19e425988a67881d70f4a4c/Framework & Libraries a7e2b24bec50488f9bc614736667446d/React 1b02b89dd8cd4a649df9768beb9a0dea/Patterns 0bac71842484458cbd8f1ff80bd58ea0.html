Controlled Components

HTML form elements that keep track of their own state need to be controlled in React in order to have only one source of truth/state.

The pattern for this consists of the following:

  1. Setting the value attribute on the form element with a variable that is stored in state.
  1. Using an onChange (camelCase, because it's React!) event that fires the setter function set a new value when the input changes.

  • this pattern relies on the event object to grab the HTML form state event.target.value is used for this.

For our purposes, we will define a Controlled List as a select list that takes user input, but is built without using any type of HTML form input (no <input>tags, no <select> tags). Both the <DayList> and the <InterviewerList> were built this way.

The thing to note here is that because these lists are built without HTML form inputs, the lists do not have built-in value or onChange attributes.

which is baked into the value attribute as its own state monitor.

Also html lists <ul> and <li> tags are easy to style, however they don’t come with a built-in value attribute or onChange event handler.

Immutable Update Patterns — hotfix

Problem:

  • If you are updating a state that is an array, Ie adding to the array, then useState actually won’t detect the change, resulting in no rendering.. this is a bug.

Solution:

  • The immutable Update Pattern is where you
    1. make a duplicate of the state
    1. add your item to it
    1. setState to the new freshly created copy
      1. use the spread operator for more elegant code:
      import React, { useState } from 'react';
import logo from './logo.svg';
import './App.css';

function App() {
  const [items, addItem] = useState(['one', 'two']);

	// old-fashioned
  const handleAddItem = () => {
    const cloneItems = items.slice(); // COPY
    cloneItems.push('three');
    addItem(cloneItems);
  }

// const handleAddItem = () => addItem([...items, 'three'])

  return (
    <div className="App">
    {items.map( item => <li>{item}</li>)}
    <button onClick={handleAddItem}>ADD ME</button>
    </div>
  );
}

The {...} spread operator

// making a copy for arrays 
const array = [1,2,3,4,5];
const copyArr = [...array];

// making a copy for objects
const obj = {a:1, b:2};
const copyObj = {...obj};

the handleItem from the example above becomes:

const handleAddItem = () => addItem([...items, 'three'])
