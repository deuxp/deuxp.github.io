controlled components

REMEMBEE to prevent the default for form submission even in React JSX

onSubmit event handler captures the event and stops it using the event.preventDefault() method

Are components in React that override HTML form elements, so that React can control their state.

The pattern involves:

  1. Set a variable that is stored in state as the value attribute on the form element.
  1. Use an onChange event that uses the setter of your state to set a new value when the input changes.

ie.,

const [name, setName] = useState('') // hook

// ...

<input value={name} placeholder="Type your name" onChange={e => setName(e.target.value)}></input>

  • THis is how you track the current input state.. you can use this now in other areas of the app

(Recall from reading about event handling that events in the DOM generate event objects which have lots of useful information for programmers. When the user types on the keyboard, that triggers the onChange event and the value of the data is found in event.target.value .)

Forms are important to a website and user interaction.

ie., <input> <textarea> <select>

However, we want React top track the state of the entire wb app, that is the whole point.

function DisplayWord(props) {
  const [word, setWord] = useState("");

  return (
    <main>
      <input
        value={word}
        onChange={(event) => setWord(event.target.value)}
        placeholder="Please enter a word"
      />
      <h1>Your word is: {word}.</h1>
    </main>
  );
}
