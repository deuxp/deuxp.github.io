Write stories to see the different contexts of the components with hard-coded data

The following code would go into your stories/index.js:

storiesOf("DayListItem", module) //Initiates Storybook and registers our DayListItem component
  .addParameters({
    backgrounds: [{ name: "dark", value: "#222f3e", default: true }]
  }) // Provides the default background color for our component
  .add("Unselected", () => <DayListItem name="Monday" spots={5} />) // To define our stories, we call add() once for each of our test states to generate a story
  .add("Selected", () => <DayListItem name="Monday" spots={5} selected />) 
  .add("Full", () => <DayListItem name="Monday" spots={0} />)
  .add("Clickable", () => (
    <DayListItem name="Tuesday" setDay={action("setDay")} spots={5} /> // action() allows us to create a callback that appears in the actions panel when clicked
  ));

  • when you work with storybook you are making a component and deconstructing the individual behaviour of one component
    • daylistitem is one component with four different behaviours depending on the state
      • you are isolating the component and then isolating the behaviours at every state to make sure they work properly at the granular level.

StoriesOf()

storiesOf is a function that takes 2 params

  1. a string that labels a group of stories.
  1. the module is a webpack module, which is available on the global (per-file) scope. If it's not included you'll need to refresh your browser with each change you make. Don't worry too much about this parameter.

appointment labels the group of stories

storiesOf("Appointment", module)

.addParameters()

The addParameters function can be chained to storiesOf and can be passed an object of parameters. We will use the backgrounds property to set the Storybook preview background colour to white.

storiesOf("Appointment", module)
.addParameters({
    backgrounds: [{ name: "white", value: "#fff", default: true }]
  })

.add()

Once this initial code has been set up, we can add stories by chaining an add() function for any number of stories that we want to write.

First, we simply want this story to render our <Appointment> component with no props. We add a new story by chaining add().

The add() function accepts the name of the story as a string and a function that returns a React component.

storiesOf("Appointment", module)
  .addParameters({
    backgrounds: [{ name: "white", value: "#fff", default: true }]
  })
  .add("Appointment", () => <Appointment />)

action()

  • allows us to create a callback that appears in the Storybook actions panel when the event happens.
  • it takes a string argument.
    • The string that is passed to action() will be displayed in the Storybook Actions panel when the component is clicked.
