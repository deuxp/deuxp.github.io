Building Complex Data Structures

Always think about the shape of the data.

Arrays vs Objects

take the time to think about the necessary steps to build data structures:

  • Determine what data you need
  • Determine what operations you need to perform on the data
  • Organize the data
  • Determine if certain data is related or independent and decide what kind of substructure is needed
  • Start small, then grow your structure

By following those steps, we're sure to always create well-formed structures that organize our data in a logical and understandable way!

Array, use if:

  1. storing things in order
    1. Problems arise when we want to create, read or modify the elements that are not at the beginning or the end of the array.
      1. or when you want to store related data.

Objeccts, use if:

  1. storing related data - order not so important
    1. easy to access and manipulate
    1. PRoblems: when order is important. How can you remove the last one?
      1. Also a problem when you have a list of individual tasks.

Consider when choosing

  1. are they independent items? - array
  1. related items? - obj
  1. access to specific values often? - obj
  1. remove the first and last element often? - array

When we create data structures, it's important to think about the nature of the data.

  • Do you have several pieces of data that are related or
  • is each piece of data independent of the others?
  • what kind of operations do you want to perform on it
