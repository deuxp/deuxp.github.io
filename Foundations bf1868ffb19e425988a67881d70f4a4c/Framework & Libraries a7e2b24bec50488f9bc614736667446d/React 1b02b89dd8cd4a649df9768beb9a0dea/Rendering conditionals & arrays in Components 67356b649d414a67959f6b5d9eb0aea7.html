ternary Operator

However, a statement using the ternary operator is allowed because it resolves to an expression.

function Hello(props) {
// CORRECT: Ternary is allowed. One of the two <h1s> will be rendered
  return (props.yourName ? <h1>Hello, {props.yourName}.</h1> : <h1>Sorry, you don't seem to have a name.</h1>);
}

The ternary operator is a good choice when you have two different possible elements to display. It can be useful to think the ternary operator like an if...else statement.

Short Circuit evaluation

The logical && operator can be used for short-circuit evaluation. It is evaluated left to right, and if the first expression is false, the second expression is not evaluated at all. Consider this:

function Hello(props) {
  return (props.yourName && <h1>Hello {props.yourName}</h1>);
}

{props.yourName} && <h1>Hello {props.yourName}</h1> is short-circuit evaluated. If props.yourName contains a value that is truthy, the <h1> will be rendered. However, if props.yourName is falsy then the JSX expression <h1>Hello {props.yourName}</h1> is not evaluated at all and nothing is shown. Any side effects of attempting to render the <h1> don't matter, and no error will be thrown.

The && operator is a good choice if you want to show something or nothing at all. It can be useful to think of the && operator like an if statement.

Classnames - Library

we use this library to cleanly handle multiple conditional className appending coming from the props.

  • A way to avoid a page of if statements..
    • many prop choices gets messy

The library gives us access to a func called: classnames:

classNames('foo', 'bar'); // => 'foo bar'

example:

classNames('foo', { bar: true }); // => 'foo bar'
classNames('foo', { bar: false }); // => 'foo'

classNames('foo', 'bar'); // => 'foo bar'
classNames('foo', { bar: true }); // => 'foo bar'
classNames({ 'foo-bar': true }); // => 'foo-bar'
classNames({ 'foo-bar': false }); // => ''
classNames({ foo: true }, { bar: true }); // => 'foo bar'
classNames({ foo: true, bar: true }); // => 'foo bar'

// lots of arguments of various types
classNames('foo', { bar: true, duck: false }, 'baz', { quux: true }); // => 'foo bar baz quux'

// other falsy values are just ignored
classNames(null, false, 'bar', undefined, 0, 1, { baz: null }, ''); // => 'bar 1'
‚ÄúIn the example above, the first argument is a string, but the second argument is an object. The string foo is always returned, but bar is a key to a boolean value. If the boolean value is true , bar is added to the string that is returned. If false, bar is not added.‚Äù

export default function Button(props) {
   const buttonClass = classNames('button', {
      'button--confirm': props.confirm,
      'button--danger': props.danger
   });
   
   return <button onClick={props.onClick} 
                  className={buttonClass}
                  disabled={props.disabled}
                  >{props.children}</button>;
}
