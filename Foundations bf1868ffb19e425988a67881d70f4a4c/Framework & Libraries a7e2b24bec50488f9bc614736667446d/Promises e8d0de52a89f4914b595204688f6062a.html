Promises

An official introductory description: A promise represents the eventual result of an asynchronous operation. The primary way of interaction with a promise is through its then method, which registers callbacks to receive either a promise’s eventual value or the reason why the promise cannot be fulfilled.

  • A promise is an object
  • Promises do not rely on anything other than basic JavaScript
  • As of ES6, JavaScript has promises supported natively in its code. In other words, they are built into the language (via Promise)
  • it fixes callback hell

Intro

Promises give us another way to interact with asynchronous code. Many of the asynchronous functions that you use on a day-to-day basis will use the Promise syntax that we covered today.

Promises let us handle values whose final value is uncertain or takes time to calculate. When a Promise is returned it's immediate value is pending. A pending value will eventually either resolve or reject depending on whether it was successful or not.

A function that returns a Promise gives us three standard functions that we can use to interact with it:

  • .then() -> waits for the promise to resolve and gives us access to the returned value
  • .catch() ->if our Promise rejects, lets us handle any errors thrown by our promise immediately
  • .finally() -> lets us run code after all of our .then and .catch functions are complete
promiseFunc()
  .then(response => {
    // do something with the response from our Promise
  })
  .catch(error => {
    // handle any errors from our Promise (including any errors from a .then())
  })
  .finally(() => {
    // will run after every .then() or .catch() is complete
		// usually want to manually close out the func if using streams etc
  })

Creating Promises

We can create our own Promises using the Promise() constructor.

// this example promise will resolve with the value "Example value" after 1s
const examplePromise = new Promise((resolve, reject) => {
  setTime(() => {
    resolve("Example value")
  }, 1000)
})

Read more about the Promise() constructor on MDN

Promise.all() and Promise.race()

Promise.all() lets us wait for multiple Promises to finish and then gives us access to all of their responses.

const promises = [promise1, promise2, promise3]
Promise.all(promises).then((values) => {
  // now we can access the values of the promises in the order we passed them into Promise.all
  // eg. the result of promise1 can be accessed as values[0], etc.
})

Promise.race() lets us run multiple Promises at the same time and returns the value of whichever Promise finished first.

const promises = [promise1, promise2, promise3]
Promise.all(promises).then((value) => {
  // now we can access the value of whichever Promise resolved first
})

Creating a new Promise

The guts of the machine ..

A promise needs two callback functions

  1. for success (resolve) and
  1. for failure (reject)

Functions can create and return promises:

/** CREATING A PROMISE **\
 * Input   -> number of dollars to loan out
 * Returns -> Promise of loan which may not fulfill, based on remaining credit
 *
 *    [] If creditLimit is less than the requested amount, only the reamaining limit is loaned out,
 *    [] otherwise the full amount is loaned out.
 *    [] If $0 remain in the limit, the loan request is rejected (error)
 */
let creditLimit = 485;

const loanOut = function(amount) {
  
  return new Promise((resolve, reject) => { // without a resolve, the script runs, but does not resolve with any money.

    if (!creditLimit) reject('insufficient funds');
    
    if (amount > creditLimit) {
      amount = creditLimit;
      creditLimit = 0;
    } else {
      creditLimit -= amount;
    }
    resolve(amount);
  });
};


// Driver Code
console.log('asking for $400, which should be okay ...');
loanOut(400)
  .then(amountRecieved => {
    console.log(`\t-> I got $${amountRecieved} loan from the bank!! Remaining Credit Limit: $${creditLimit}`);
  })
  .catch(err => {
    console.log(`\t-> Error: ${err}!`);
  });

These promises can have logic which then decides how to resolve or reject.. calling the appropriate callback, .then or .catch, and passing in the relevant data to these callbacks.

The caller can then handle these situations and the data that comes back from the promise.
