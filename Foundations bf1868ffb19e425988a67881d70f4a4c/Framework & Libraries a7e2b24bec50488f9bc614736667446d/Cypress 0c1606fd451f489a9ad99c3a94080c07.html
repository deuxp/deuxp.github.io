.intercept()

to use your own reproducable data

the second arg is a very fuzzy match for the WEB ADDRESS

the third arg is what you are replacing it with, which is one of the fixture files

It is promise based (cy) so you can assign an alias to the intercept, get request, and then use cy.wait(’@alias’) then after that run the rest of your checks, (dont need to chain it with .then, just wait for the alias to resolve and then the wait function will take care of the waiting.)

template

describe("#Appointments", () => {
  beforeEach(() => {
    // Visits the root of our web server
    cy.request("GET", "/api/debug/reset");
    cy.visit("/");
    cy.contains("Monday");
  });

  it("should book an appointment", () => {
    cy.get('[alt="Add"]').first().click();
    cy.get("input").type("Lydia Miller-Jones");
    cy.get('[alt="Tori Malcolm"]').click();
    cy.contains("Save").click();
    cy.contains(".appointment__card--show", "Lydia Miller-Jones");
    cy.contains(".appointment__card--show", "Sylvia Palmer");
  });
  it("can edit an existing appointment", () => {
    cy.get('[alt="Edit"]').click({ force: true });
    cy.get('[data-testid="student-name-input"]').type(
      "{selectAll}Friedi Khalo"
    );
    cy.get('[alt="Tori Malcolm"]').click();
    cy.contains("Save").click();
    cy.contains(".appointment__card--show", "Friedi Khalo");
    cy.contains(".appointment__card--show", "Tori Malcolm");
  });
  it.only("should cancel an existing appointment", () => {
    cy.get('[alt="Delete"]').click({ force: true });
    cy.contains("Confirm").click();
    cy.contains("Deleting").should("exist");
    cy.contains("Deleting").should("not.exist");
  });
});
  • contains is good for words that you see or css class selectors
  • get is good for computed attributes or testid
  • use should exist or not exist for confirming changing looks. This takes advantage of Cypress natural async waiting.
  • use force:true to click hidden items
  • reset the test db using beforeEach
