Insight

The details about the library we use for making the request don't matter until we answer the following questions:

  • When should we request the data?
  • Where should we store the retrieved data?
  • What changes cause us to make the request again?

The four most common methods:

import axios from 'axios'

axios.get(url[, config])
axios.post(url[, config])
axios.put(url[, config])
axios.delete(url[, config])

The Pattern (HTTP)

ie., Simple example:

axios.get("/api/appointments").then((response) => {
  console.log(response);
});

Returns a response in the form of a promise.

When you want to send data to the RESTful server, ie put post or delete, use the second argument to specify this information {} in the form of an object.

axios
  .put(`/api/appointments/2`, {
    id: 2,
    time: "1pm",
    interview: {
      student: "Archie Cohen",
      interviewer: 9,
    },
  })
  .then((response) => {
    console.log(response);
  });

handle errors with a catch. Include response code:

axios
  .get("/api/appointments")
  .then((response) => {
    console.log(response);
  })
  .catch((error) => {
    console.log(error.response.status);
    console.log(error.response.headers);
    console.log(error.response.data);
  });

401: login with incorrect credentials

403: user accessisng a resource without permission

we can make multiple requests at the same time for our dependent data. We used the Promise.all function to run many promises concurrently and when all the Promises resolved, we updated the state.
