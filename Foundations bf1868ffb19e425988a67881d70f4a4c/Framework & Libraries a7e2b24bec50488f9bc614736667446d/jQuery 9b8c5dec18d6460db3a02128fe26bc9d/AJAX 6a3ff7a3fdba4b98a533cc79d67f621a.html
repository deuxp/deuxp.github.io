AJAX

ajax is used to get data from web servers into strings in javascript.. Usually these strings are JSONs; and this pattern is so common that jQuery automatically parses the JSONs if they are detectable as such.

$(function() {
  const $button = $('#load-more-posts');
  $button.on('click', function () {
    console.log('Button clicked, performing ajax call...');
    $.ajax('more-posts.html', { method: 'GET' })
    .then(function (morePostsHtml) {
      console.log('Success: ', morePostsHtml);
      $button.replaceWith(morePostsHtml);
    });
  });
});

JQ has its own ajax method

  • first the filename or information target is passed as the first arg
  • second arg is the HTTP method as an object { method: GET }
    • now the request is made, it is like a promise.. it uses the .then chain to do something with the recieved ajax call
      • the callback is an anonymous function with the variable you give as a name as an alias for the returned data (it actually looks wierd because it is not written as a fat arrow function.. which is totally valid, unless you are explicitly referencing `this` anywhere in the scope of the callback)
    • then the button which is in a JQ wrapper is being replaced with the new data,
      • which is a straight HTML string literal which will be parsed as such, since it is replacing the button html representation in the DOM tree

ajax request: define the header: method, data type, url → which is the route

and then what happens when there is a success, which is basically like a promise.handler.

const fetchPosts = () => {
    $.ajax({
      method: "GET",
      dataType: "json",
      url: "/api/posts",
      success: posts => {
        console.log(posts)
        createPosts(posts)
      },
      error: error => console.error(error)
    })
  }

JQuery AJAX shorthands

// make a get request to the specified endpoint
$.get("https://jsonplaceholder.typicode.com/posts");

// make a get request for JSON data
$.getJSON("https://jsonplaceholder.typicode.com/posts");

// make a post request
$.post("https://jsonplaceholder.typicode.com/posts", {
  /* form data */
});

The server is configured to receive form data formatted as a query string. The exact structure of query strings is not standardized, but a common format is a set of field-value pairs, where the field is separated from the value by = and each pair is separated from the next pair by &

field1=value1&field2=value2&field3=value3...

The jQuery .serialize() function turns a set of form data into a query string. This serialized data should be sent to the server in the data field of the AJAX POST request.
