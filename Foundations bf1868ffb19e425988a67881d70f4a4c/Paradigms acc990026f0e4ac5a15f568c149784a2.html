A function must have (a) at least one recursive case and (b) at least one base case in order to be recursive

  • Every recursive function must have a base case and a recursive case.
  • Each recursive call should break down the current problem into a smaller, simpler one.
  • The recursive calls must eventually reach the smallest version of the problem, the base case.
  • The base case is when the problem can be solved without further recursion.

It is an alternative to for and while loops.

everything you can do with a loop, you can do recursively as well.

Unit Testing

Using external resources like a network or database is not a part of unit testing.

  • give the function some inputs
  • check the output for correctness

regressions - bugs that occur repeatedly

  • unit tests can prevent this
  • test driven design can help you improve your code design

Test how parts of the system work together - the integreation of the parts.

  • similar to unit tests but not isolated from each other.
  • example would be a database and an app working together. over an network

Async control flow

The web is asynchronous

an asynchronous function puts it in the event loop.. everything that interacts with this async function must be nested in the original async call.. Data returned is dealt with via a callback. Return is only used in async logic to end the control flow. Not to return any information used by the asynchronous function.

Sync

const breedDescription = fetchBreedDescription('Siberian');

Async

fetchBreedDescription('Siberian', (error, description) => {

});

  • that said it is not the funciton itself that is asynchronous.. it is the TASK that it performs, like loading or waiting for a timer.
    • for example., having EventListener()s alone does not make code asynchronous.
      • an event listener waiting for a click to happen is waiting.. but its not doing anything until a click happens. So that itself is not asynchronous. Its when a function sis performing a task that takes a while. so its thrown into an async loop to not block the rest of the script from running.

JS has syncronous APIs and asynchronous APIs

sync is just working down the file in a linear fasion.. which will have performance issues if one operation takes a long time to complete or has to wait for an event.

JavaScript leverages callbacks in order to deal with time consuming tasks. Although its still just does one thing at a time.

Node doesn't wait for code to execute before it moves on to the next bit.

It is a very common workflow in software development, expecially in Node and JS.. Callbacks are a common way to handle async flow.
